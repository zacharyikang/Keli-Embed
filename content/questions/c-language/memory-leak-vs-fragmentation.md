answer: "核心区别：\n1. 内存泄漏（Memory Leak）：指动态分配的内存（如 malloc）在使用完毕后没有释放，导致这部分内存被永久占用，系统可用内存越来越少，最终可能崩溃。\n2. 内存碎片（Memory Fragmentation）：指频繁分配和释放不同大小的内存，导致内存空间中出现许多细小且离散的空闲块。即使总的空闲内存足够，但也无法满足大块内存的连续分配需求。\n实际编程中的预防措施：\n1. 预防内存泄漏：尽量使用静态分配（如全局变量、栈内存）；如果必须用动态内存，遵循“谁分配谁释放”原则；在小型系统中使用内存池管理。\n2. 预防内存碎片：尽量使用固定大小的内存块分配，避免大小混杂；减少频繁的分配和释放（一次性分配，长期使用）；采用内存池技术分类管理。"
explanation: "在资源受限的嵌入式实时系统（RTOS）或裸机中，内存泄漏和内存碎片是导致系统长期运行死机的主要杀手。因此，许多高可靠性嵌入式项目（如飞控、BMS等）直接禁止使用标准库的 malloc 和 free，而是通过静态缓冲区或专用的两级内存区（如 FreeRTOS 的 heap4.c/heap5.c）来实现可预测的内存分配。"
