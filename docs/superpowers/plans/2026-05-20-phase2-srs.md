## 算法设计

### 简化 SM-2

与经典 SM-2 的偏差（按 spec §4.3）：

| 维度 | 经典 SM-2 | EmbedStudio |
|------|-----------|-----------|
| 评分等级 | 0-5 (6 级) | again/hard/good/easy (4 级) |
| EF 微调公式 | 复杂公式 | 线性 ±0.15 |
| Now 注入 | Date.now() 内部 | now: Date 参数注入 |

### schedule(state, rating, now)

**again (忘了):**
- EF = max(1.3, EF - 0.2)
- intervalDays = 0, repetitions = 0
- dueAt = now + 10min

**hard / good / easy (答对):**
- EF = max(1.3, EF + delta)  // delta: hard=-0.15, good=0, easy=+0.15
- Interval:
  - repetitions=0 → 1 day
  - repetitions=1 → 3 days
  - else → round(prevInterval × newEF)
- hard modifier: interval = max(1, round(interval × 0.8))
- repetitions + 1

始终返回新对象（不修改输入）。

### buildTodayQueue

三层优先级：
```
Priority 0: Weak cards (按 weakMarkedAt ASC)
Priority 1: Due cards  (按 dueAt ASC)
Priority 2: New cards  (direction round-robin)
→ slice(0, dailyTotalLimit)
```

输入类型：
```typescript
type BuildTodayQueueOptions = {
  weakCards: CardState[];
  dueCards: CardState[];
  newCardsCandidates: Question[];
  dailyNewLimit: number;     // 默认 10
  dailyTotalLimit: number;   // 默认 50
};
```

输出类型：
```typescript
type ScheduledCard = {
  card: CardState;
  priority: 0 | 1 | 2;
  question?: Question;      // 仅 priority=2 时有
};
```

### pickNewCards(candidates, limit)

1. 按 direction 分组
2. 同方向按 difficulty 排序：easy → medium → hard
3. direction round-robin 轮转选
4. 候选池不足时返回已选部分

### markAsWeak(state|null, questionId, now)

- state 为 null 时创建 emptyCardState
- 设 isWeak=true, weakMarkedAt=now, dueAt=now
- **不修改** easeFactor / intervalDays / repetitions / lastRating

### maybeUnmarkWeak(state, newRating)

- state.isWeak=false → return state
- 连续 2 次 good-or-better → 清除 isWeak 标记
- 否则 return state

## 纯函数纪律

- 所有函数都是 `(input) → output`，无副作用
- `now: Date` 从外部传入，不调 `Date.now()`
- 不 import React / Next / Supabase
- 可遍历所有评分组合 + 边界值
